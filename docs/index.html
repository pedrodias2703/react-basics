Redux

Before going into Redux, I must talk about Flux.

Flux is an architecture used for managing data flow in your application. The most important concept is that data flows in one direction.

MVC Flux

Flux's pieces:

  • Dispatcher
  • Store
  • Action
  • View
  • Dispatcher: The dispatcher receives actions and dispatches them to stores that have registered with the dispatcher. Every store will receive every action. There should be only one singleton dispatcher in each application.
  • Store: A store is what holds the data of an application. Stores will register with the application's dispatcher so that they can receive actions. The data in a store must only be mutated by responding to an action. There should not be any public setters on a store, only getters. Stores decide what actions they want to respond to. Every time a store's data changes it must emit a "change" event. There should be many stores in each application.
  • Action: Actions define the internal API of your application. They capture the ways in which anything might interact with your application. They are simple objects that have a "type" field and some data.
  • View: Data from stores is displayed in views. REACT COMPONENTS!

Redux is just an implementation of Flux architecture.

Concepts

  • To change something in the state, you need to dispatch an ACTION, a plain JavaScript object that describes what happened.
  • To tie state and actions together, we write a function called a REDUCER, a function that takes state and action as arguments and returns the next state of the app.
  • The main idea is that you describe how your state is updated over time in response to action objects

Routes

Basically, routes on React are all about displaying the right component depending on the URL.

It can be as simple as:


                  if (window.location.pathname === '/about') {
                    return <About />
                  } else {
                    return <App />>
                  }

But to add a condtion to every URL that we want to mange would be extremely boring

That's why there are a lot of libraries to handle routing on React

The most widely used is React Router

https://reacttraining.com/react-router/

Its philosophy is to achieve dynamic routing rather than using static routes. Let's take a look at the differences between then.

In Express, you would declare your routes like:


                    app.get('/users');
                    app.get('/users/:id');
                    app.get('users/:id/edit');
                    
                    app.listen();

Note how the routes are declared before the app listens.

The client side routers we’ve used are similar. In Angular you declare your routes up front and then import them to the top-level AppModule before rendering:


                      const appRoutes: Routes = [
                      { path: 'users',
                      component: UsersListComponent
                    },
                    { path: 'user/:id',
                    component: UserDetailComponent
                  }
                  ];							  
                  @NgModule({
                    imports: [
                    RouterModule.forRoot(appRoutes)
                    ]
                  })							  
                  export class AppModule { }

In dynamic routing, routing that takes place as your app is rendering, not in a configuration or convention outside of a running app. That means almost everything is a component in React Router.


                    import { BrowserRouter, Route, Link } from 'react-router-dom';
                    
                    ReactDOM.render((
                    <BrowserRouter>
                    <App/>
                    </BrowserRouter>
                    ), el);
                    
                    const  App = () => (
                    <div>
                    <nav>
                    <Link to="/dashboard">Dashboard</Link>
                    </nav>
                    <div>
                    <Route path="/dashboard" component={Dashboard}/>
                    </div>
                    </div>
                    )

The Route will render <Dashboard {...props}/> where props are some router specific things that look like { match, location, history }. If the user is not at /dashboard then the Route will render null.

Routers

At the core of every React Router application should be a router component. 


                      import { BrowserRouter as Router } from 'react-router-dom'
                      ReactDOM.render((
                      <Router>
                      <App/>
                      </Router>
                      ), holder)

Tip: a <Router> can only have one child.

Route Matching

Route matching is done by comparing a <Route>'s path prop to the current location’s pathname. When a <Route> matches it will render its content and when it does not match, it will render null. A <Route> with no path will always match.

You can include a <Route> anywhere that you want to render content based on the location

Route Rendering Methods

You have three prop choices for how you render a component for a given <Router>

  • component: should be used when you have an existing component (either a React.Component or a stateless functional component) that you want to render.
  • render: takes an inline function, should only be used when you have to pass in-scope variables to the component you want to render.
  • children: works exactly like render except that it gets called whether there is a match or not.

Networking

As we know, React it's nothing more than a Javascript library. Fetching a resource through a network is done the same way any Javascript client would do.

Options:

  • XMLHttpRequest
  • jQuery.ajax
  • Fetch API
  • Axios
  • ...etc

There are advantages and disadvantages between all the options, but some are much more noticeable. For that reason, we will use Axios on our example.

The trick part about fetching a resource on the network with React is to decide where in the component lifecycle you will place you AJAX request.

React documentation tells us to make AJAX inside the componentDidMount method so we can use setState to update our component when the data is retrieved. That way, we have more control over the states of our component.

Let's code!

We will use the Star Wars API, which is a free API for testing and learning purposes.

https://swapi.co/

Let's first create a new component:


                      // components/starwars/films.js
                      import React, { Component } from 'react';
                      class Films extends Component {
                        constructor(props) {
                          super(props);
                        }
                        render() {
                          return <div>Hi!</div>
                        }
                      }
                      export default Films;

Now let's install the Axios package:


                        npm i --save axios

And import Axios and define an initial state for our component:


                      // components/starwars/films.js
                      import axios from 'axios';

                      constructor(props) {
                        super(props);
                        this.state = {
                            loading: false,
                            error: false,
                            list: []
                        }
                      }

Define a function that will fetch our data:


                      fetchFilms = () => {
                        this.setState({ loading: true, error: false }, () => {
                            axios.get('https://swapi.co/api/films/')
                                .then((response) => {
                                    const list = response.data.results;
                                    this.setState({
                                        loading: false,
                                        error: false,
                                        list
                                    });
                                })
                                .catch(() => {
                                    this.setState({
                                        loading: false,
                                        error: true,
                                    });
                                });
                        });
                      }

Like we learn before, the place to fetch our data is inside componentDidMount:


                      componentDidMount() {
                        this.fetchFilms();
                      }

Define the render method:


                      render() {
                        const { loading, error, list } = this.state;
                        if (loading) { return  <div>Loading...</div>; }
                        if (error) { return <div>Error!</div>; }
                        return (
                          <div>
                            <ul> {
                              list.map( film => (
                                  <li key={film.episode_id}>{film.title}</li>
                              ))
                            }
                            </ul>
                          </div>
                        );
                      }

Change our App.js to render our new component:


                      // App.js

                      <Link to='/films'>Films</Link>

                      <Route path="/films" component={Films}/>
